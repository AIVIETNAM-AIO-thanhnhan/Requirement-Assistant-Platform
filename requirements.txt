# Requirement Assistant Platform
#
# Python:
# Python 3.10 or Python 3.11 recommended
#
# Install:
# pip install -r requirements.txt
#
# This project supports:
#
# Embedding Providers
# -------------------
#
# Option A — Ollama Embeddings (local)
# Model: nomic-embed-text
#
# Option B — BGE Embeddings (local)
# Model: BAAI/bge-base-en-v1.5
#
# Option C — OpenAI Embeddings (cloud)
# Model: text-embedding-3-small
#
# LLM Providers
# -------------
#
# Option A — Ollama (local)
# Free
# No API key
# Recommended for beginners
#
# Option B — OpenAI (cloud)
# Paid API
# Better answer quality
#
# Configure in .env:
#
# EMBED_PROVIDER=ollama | bge | openai
# LLM_PROVIDER=ollama | openai
#
# Recommended Configurations
# --------------------------
#
# Fully Local Setup
#
# EMBED_PROVIDER=ollama
# OLLAMA_EMBED_MODEL=nomic-embed-text
#
# LLM_PROVIDER=ollama
# OLLAMA_LLM_MODEL=llama3.2
#
# Better Retrieval Quality (Recommended)
#
# EMBED_PROVIDER=bge
# BGE_EMBED_MODEL=BAAI/bge-base-en-v1.5
#
# LLM_PROVIDER=ollama
# OLLAMA_LLM_MODEL=llama3.2
#
# Explanation:
#
# BGE creates embeddings only.
#
# BGE is NOT an LLM.
#
# If using BGE:
#
# EMBED_PROVIDER=bge
#
# you still need:
#
# LLM_PROVIDER=ollama
#
# or:
#
# LLM_PROVIDER=openai
#
# Pipeline:
#
# User Question
# ↓
# BGE Embedding
# ↓
# ChromaDB Retrieval
# ↓
# Ollama / OpenAI
# ↓
# Final Answer
#
# Production / Best Quality
#
# EMBED_PROVIDER=openai
# OPENAI_EMBED_MODEL=text-embedding-3-small
#
# LLM_PROVIDER=openai
# OPENAI_LLM_MODEL=gpt-4o-mini
#
# Benchmark Comparison
#
# Ollama:
# nomic-embed-text
#
# BGE:
# BAAI/bge-base-en-v1.5
#
# OpenAI:
# text-embedding-3-small
#
# Run:
#
# python scripts/benchmark_embeddings.py
#
# Metrics:
#
# Accuracy (Pairwise Retrieval Accuracy)
# Explanation: For each query, we have a known relevant case (positive) and a known irrelevant case (negative). The embedding model calculates:
#       Similarity(Query, Positive)
#       Similarity(Query, Negative)
#   PASS if Similarity(Query, Positive) > Similarity(Query, Negative)
# Recall@1: Cases Found In Top  1
# Recall@3: Cases Found In Top 3
# Note: For RAG systems, top 3 is often more important than top 1, as the LLM can still generate a good answer if the relevant case is among the top 3 retrieved.
# Recall@5: Cases Found In Top 5
# MRR (Mean Reciprocal Rank): How HIGH the correct chunk is ranked = Average of all reciprocal ranks
# Latency
# Explanation: Time taken for each step of the pipeline:
#   Embedding: Time to generate embedding
#   Retrieval: Time to retrieve from vector database
#   LLM: Time to generate answer
#
# For a RAG project demo, Recall@3 and MRR are the two numbers that most convincingly show retrieval quality.
#
# Outputs:
#
# outputs/embeddings/reports/
# outputs/embeddings/images/
#
# Run:
#
# python scripts/benchmark_llms.py
#
# Outputs:
#
# outputs/llms/reports/
# outputs/llms/images/
#
